# 嘀果 (Diggo) 前端机试题

## 欢迎
感谢您参与嘀果 (Diggo) 的前端工程师面试。本仓库包含前端机试题目素材及要求。请严格阅读以下说明进行答题。

## 答题要求
1. **技术栈限制**：仅限使用 **HTML5 + CSS3** 实现。
   - **禁止**使用：JavaScript (JS)、Sass/SCSS、Bootstrap、Tailwind 等第三方库或预处理器。
   - 必须保持代码纯净。
2. **还原度**：要求像素级还原效果图，细节还原度需达到 90% 以上。
3. **命名规范**：CSS 类名需语义化（推荐使用 BEM 命名规范），代码结构清晰。
4. **字体说明**：
   - 列表标题 (`Latest Update Apps`)：使用 ProductSans 字体。
   - 其余文字部分：使用 Roboto 字体。
   - （请自行在 CSS 中通过 @font-face 引入题目提供的字体文件）

## 提交流程
1. **Fork 仓库**：请将本仓库 Fork 到您的个人 GitHub 账户下。
2. **环境设置**：在本地 `/Interview/web/` 目录下完成答题。
3. **代码提交**：
   - 请保持文件目录结构与模板一致。
   - 提交内容仅限 `index.html` 和 `style.css` 及必要的素材（禁止上传 node_modules 或不需要的编译文件）。
4. **完成提交**：请以 **【姓名】** 命名您的仓库分支，并完成代码 commit。
5. **发送结果**：将您的简历 PDF 及代码仓库地址（或 zip 打包文件）通过邮件发送至邮箱 zhaopin@diggo-inc.com。

## 题目说明

### 题目一 (40 分)
- 实现效果图中的列表布局。
- 注意：页面宽度为 360px。
- App 标题展示为两行，溢出部分需以省略号显示。
- 必须实现 HTML 标签语义化。
- **请务必在完成后截图，重命名为 `result.png` 存放在根目录。**

### 题目二 (60 分)
在题目一基础上实现响应式布局，要求：
- 页面宽度 ≥ 768px 时：每行展示两个 App。
- 页面宽度 ≥ 1024px 时：每行展示三个 App。
- 桌面端正文内容须保持**居中展示**。
- App Logo 图片和下载按钮中间的宽度需**自适应**。
- **注意**：请严格参考设计稿标注尺寸，而非视觉估算尺寸。
- 建议：CSS 媒体查询控制在两个以内。
